# MSG - 알림 자동 전달

스마트폰에 오는 알림을 자동으로 문자(SMS), URL, Telegram, Discord, Slack으로 전달해주는 앱입니다.
특정 앱의 특정 키워드가 포함된 알림만 골라서 전달할 수 있습니다.

---

## 이런 분들에게 유용합니다

- 카드 결제 알림을 다른 번호로 자동 전달하고 싶은 분
- 택배 알림을 받으면 자동으로 웹훅(URL)을 호출하고 싶은 분
- 특정 앱 알림을 SMS로 받아보고 싶은 분
- 업무 알림을 외부 시스템(Slack, 텔레그램 봇 등)에 자동 연동하고 싶은 분
- 알림을 텔레그램 봇으로 받아보고 싶은 분

---

## 주요 기능

### 알림 감시 및 저장
- 스마트폰의 모든 앱 알림을 실시간으로 감시
- 알림 제목, 내용, 이미지까지 저장
- 날짜별로 알림 기록 확인 가능
- 30일 지난 기록은 자동 삭제

### 조건 기반 자동 전달
- **앱 선택**: 감시할 앱을 지정 (예: 카카오톡, 신한카드 등)
- **키워드 필터**: 제목/내용에 특정 단어가 포함될 때만 전달
- **AND/OR 모드**: 모든 키워드 일치 또는 하나만 일치해도 전달
- **정규식 지원**: 복잡한 패턴 매칭도 가능

### 전달 방식
- **SMS 전달**: 지정한 번호로 자동 문자 발송
- **URL 호출**: GET/POST 방식으로 외부 API 호출
- **Telegram 봇**: 텔레그램 봇을 통한 메시지 전송
- **Discord 웹훅**: Discord 채널로 자동 메시지 전송
- **Slack 웹훅**: Slack 채널로 자동 메시지 전송
- SMS, URL, Telegram, Discord, Slack을 동시에 사용 가능

### 메시지 템플릿
전달할 메시지를 자유롭게 구성할 수 있습니다:
- `{app}` - 알림을 보낸 앱 이름
- `{title}` - 알림 제목
- `{content}` - 알림 내용
- `{time}` - 알림 수신 시간

**예시:** `[{app}] {title}: {content} ({time})`

### 규칙 템플릿 (프리셋)
- 새 규칙 추가 시 용도별 템플릿 선택 가능
- **빈 규칙**: 처음부터 직접 설정
- **택배 알림**: 배달완료/픽업/도착 키워드 사전 설정
- **결제 알림**: 결제/승인/출금/이체 키워드 사전 설정
- **보안 알림**: 인증/로그인/보안/코드 키워드 사전 설정
- **서버 알림**: Discord+Slack 전송, 서버 모니터링용 사전 설정

### 통합 백업/복원
- 전달 규칙, 앱 제외, 키워드 제외를 하나의 JSON 파일로 백업
- 기기 변경 시 백업 파일 하나로 모든 설정 간편 복원

### 전달 이력
- SMS/URL/Telegram/Discord/Slack 전송 성공/실패 기록 확인
- 전송 방식별 필터링
- 실패한 전송 재시도 버튼
- CSV 파일로 내보내기

### 알림 기록 검색
- 알림 제목, 내용, 앱 이름으로 실시간 검색
- 앱별 필터와 동시 사용 가능

### 홈 화면 위젯
- **상태 위젯 (4x1)**: 서비스 상태, 오늘 수신/발송/규칙 수 한눈에 확인
- **규칙 통계 위젯 (3x1)**: 특정 규칙의 발송 횟수/성공률/마지막 발동 표시
- 위젯 추가 시 표시할 규칙 선택 가능
- 위젯 탭으로 앱 바로 실행

### 전달 일시정지
- 홈 화면에서 원터치로 전달 일시정지/재개
- 일시정지 중에도 알림 수신은 계속됨
- 규칙 매칭 및 발송만 중단

### 7일 추이 그래프
- 홈 화면에서 최근 7일간 수신/전달 추이 바 차트 확인
- 일별 알림 수신 건수와 전달 건수를 시각적으로 비교

### 시간대별 다크모드
- 설정에서 "예약" 모드 선택
- 다크모드 시작/종료 시간 설정 (예: 22:00 ~ 07:00)
- 설정 시간에 자동으로 다크모드 전환

### 제외 키워드 (NOT 필터)
- 특정 키워드가 포함된 알림은 전달에서 제외
- 제목/내용 각각 별도 설정 가능

### 테스트 발송
- 규칙 편집 화면에서 테스트 메시지를 바로 발송
- 설정이 올바른지 저장 전에 확인 가능

### 규칙 복제
- 기존 규칙을 복사하여 빠르게 유사 규칙 생성

### 규칙 통계
- 규칙별 발송 횟수, 성공률, 마지막 발동 시간 표시

### 중복 발송 방지
- 규칙별 0~300초 쿨다운 설정 가능
- 동일한 알림에 대해 설정된 시간 내 중복 발송 자동 차단

### 시간대별 규칙 활성화
- 규칙별 활성 시간대 설정 가능 (예: 09:00 ~ 18:00)
- 야간 시간대 (예: 22:00 ~ 06:00) 지원
- 비워두면 24시간 항상 활성

### 요일별 스케줄
- 규칙별 활성 요일 선택 (월~일)
- 선택하지 않으면 매일 활성
- 시간대 설정과 함께 사용 가능

### 다중 수신자 SMS
- 줄바꿈으로 여러 전화번호 입력
- 각 번호로 개별 SMS 발송
- 연락처에서 선택 시 기존 번호에 추가

### 전달 실패 자동 재시도
- 전송 실패 시 15분 주기로 자동 재시도
- 1시간 이내 실패건에 대해 최대 3회 재시도
- SMS, URL, Telegram, Discord, Slack 모두 지원

### 규칙 우선순위
- 규칙 목록에서 위/아래 화살표로 실행 순서 변경
- 높은 우선순위 규칙이 먼저 매칭

### 금액 필터
- 알림에서 금액을 자동 추출 (₩, 원, KRW, 콤마숫자)
- 최소/최대 금액 범위 설정 가능
- 예: 1만원 이상 결제 알림만 전달

### 이미지 첨부 전달
- 알림에 포함된 이미지를 Telegram/Discord로 함께 전달
- 이미지가 없으면 텍스트만 전달

### 자동 업데이트 안내
- 앱 실행 시 새 버전이 있으면 업데이트 안내 표시
- Play Store, ONE Store 등 설치한 스토어에 맞게 자동 이동

### 설정 화면
- 데이터 관리: 통합 백업·복원 (규칙+앱제외+키워드제외), 전달 이력 내보내기·삭제
- 프리미엄: 광고 제거 코드 입력/해제
- 앱 정보: 버전, 개인정보처리방침, 문의하기

### 프리미엄 (광고 제거)
- 인증 코드 입력으로 배너 광고와 앱 오프닝 광고 제거

### 연락처 선택
- 수신번호 입력 시 전화번호부에서 바로 선택 가능
- 전화번호 010-1234-5678 형식으로 자동 표시

### 리워드 광고
- 전달 규칙 3개까지 무료
- 4개째부터는 짧은 광고 시청 후 생성/수정 가능

### 다국어 지원
- 한국어 (기본)
- English

### 알림에서 규칙 추가
- 알림 기록에서 항목을 길게 눌러 상세 팝업 표시
- "규칙추가" 버튼으로 해당 알림 정보가 미리 입력된 새 규칙 생성

### 알림 제외
- **앱 단위 제외**: 특정 앱의 알림을 기록에서 완전히 제외
- **키워드 제외**: 제목/내용에 특정 키워드가 포함된 알림을 기록에서 제외
  - AND/OR 매칭 모드 선택
  - 정규식(Regex) 지원
  - 제목 키워드와 내용 키워드를 각각 독립 설정
- 알림 기록에서 길게 눌러 "알림제외" 또는 "키워드 제외"를 바로 선택 가능
- 키워드 제외 시 해당 알림의 제목/내용이 자동으로 미리 입력됨

## 사용 방법

### 1단계: 초기 설정
1. 앱을 설치하고 실행합니다
2. **알림 접근 권한**을 허용합니다 (필수)
   - 홈 화면의 "알림 권한 설정" 버튼을 눌러 설정 화면으로 이동
   - MSG 앱을 찾아 활성화
3. **SMS 발송 권한**을 허용합니다 (SMS 전달 사용 시)
4. **배터리 최적화 예외**를 설정합니다 (앱 안내에 따라)
5. 홈 화면에서 **"서비스 시작"** 버튼을 누릅니다

### 2단계: 전달 규칙 만들기
1. 하단 **"전달규칙"** 탭으로 이동
2. 오른쪽 아래 **+** 버튼을 눌러 새 규칙 생성
3. **대상 앱** 선택 (알림을 감시할 앱)
4. **키워드** 입력 (선택사항 - 비워두면 해당 앱의 모든 알림 전달)
   - 제목 키워드: 알림 제목에서 검색
   - 내용 키워드: 알림 내용에서 검색
   - 여러 키워드는 줄바꿈으로 구분
5. **전달 방식** 설정:
   - SMS: 수신 전화번호 (줄바꿈으로 여러 번호 입력 가능) + 메시지 템플릿 입력
   - URL: GET/POST 선택 + URL 주소 입력
   - Telegram: Bot Token + Chat ID + 메시지 템플릿 입력
   - Discord: Webhook URL + 메시지 템플릿 입력
   - Slack: Incoming Webhook URL + 메시지 템플릿 입력
6. (선택) **제외 키워드**: 특정 단어가 포함된 알림을 제외
7. (선택) **쿨다운**: 중복 발송 방지 시간 조절 (0~300초)
   - (선택) **활성 요일**: 특정 요일에만 규칙 활성화
8. **테스트 발송** 버튼으로 설정 확인
9. 저장하면 즉시 작동 시작

### 3단계: 확인
- **홈 탭**: 오늘 수신 알림 수, 매칭 발송 수 확인
- **알림기록 탭**: 수신된 알림 목록 확인 (길게 눌러 상세 보기)

## URL 호출 활용 예시

### GET 방식
```
https://api.example.com/notify?title={title}&content={content}&time={time}
```

### POST 방식
- URL: `https://hooks.slack.com/services/xxx/yyy/zzz`
- Body:
```json
{"text": "[{app}] {title}\n{content}"}
```

### Telegram 봇 설정 방법
1. 텔레그램에서 **@BotFather**에게 `/newbot` 전송
2. 봇 이름과 username 입력 후 **Bot Token** 복사
3. 생성된 봇에게 아무 메시지 전송
4. `https://api.telegram.org/bot<토큰>/getUpdates` 접속하여 **Chat ID** 확인
5. 규칙 편집 화면에서 Bot Token과 Chat ID 입력

### Discord 웹훅 설정 방법
1. Discord 서버 설정 → **연동** → **웹후크** 선택
2. **새 웹후크** 만들기
3. 메시지를 보낼 채널 선택
4. **웹후크 URL 복사** 클릭
5. 규칙 편집 화면에서 Webhook URL 입력

### Slack 웹훅 설정 방법
1. [Slack API](https://api.slack.com/apps)에서 앱 생성
2. **Incoming Webhooks** 활성화
3. **Add New Webhook to Workspace** 클릭
4. 메시지를 보낼 채널 선택 후 허용
5. 생성된 **Webhook URL** 복사
6. 규칙 편집 화면에서 Webhook URL 입력

## 자주 묻는 질문

**Q: 앱을 종료해도 알림이 전달되나요?**
A: 네, 포그라운드 서비스가 실행 중이면 앱을 닫아도 계속 동작합니다. 홈 화면에서 서비스 상태를 확인하세요.

**Q: 폰을 재부팅하면 어떻게 되나요?**
A: 자동으로 서비스가 다시 시작됩니다.

**Q: 특정 앱 알림을 기록하고 싶지 않아요.**
A: "알림제외" 탭에서 해당 앱을 추가하거나, 알림기록에서 항목을 길게 눌러 "알림제외"를 선택하세요.

**Q: 데이터를 백업할 수 있나요?**
A: 설정 → 데이터 관리에서 "백업" 버튼으로 전달 규칙, 앱 제외, 키워드 제외를 하나의 JSON 파일로 저장하고, "복원"으로 불러올 수 있습니다.

**Q: 특정 키워드가 포함된 알림을 아예 기록하지 않게 할 수 있나요?**
A: "알림제외" 탭에서 키워드 제외 규칙을 추가하세요. 제목/내용 키워드를 설정하면 해당 알림이 기록되지 않습니다. 알림기록에서 항목을 길게 누른 후 "키워드 제외"를 선택하면 더 편리합니다.

**Q: 배터리 소모가 심하지 않나요?**
A: 알림 리스너는 시스템에서 제공하는 기능으로, 추가 배터리 소모는 거의 없습니다.
